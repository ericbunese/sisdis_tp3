RELATÓRIO

O objetivo deste trabalho prático é implementar o algoritmo de Broadcast no VCube, sobre o ambiente de execução do SMPL (Simulation Programming Language)
Neste trabalho, consideramos apenas falhas do tipo CRASH, e também a premissa dos algoritmos: 'Apenas um evento ocorre de cada vez.'

De modo a facilitar a configuração de eventos no trabalho, o programa tempo.c espera um segundo argumento além do número de nodos.
Este segundo argumento é o nome de um arquivo de entrada, que descreverá todos os eventos que o usuário desejar escalonar
no programa, através de uma sintaxe muito simples.

Exemplo:
F,1,61.0
R,1,91.0
B,0,121.0
Na linha 1, é escalonado um evento FAULT para o nodo 1 no tempo 61.0.
Na linha 2, é escalonado um evento REPAIR para o nodo 1 no tempo 91.0.
Na linha 3, é escalonado um evento BROADCAST para o nodo 0 no tempo 121.0
Este evento BROADCAST indica que o nodo escolhido deve iniciar a rodada de difusão de informações.

Neste trabalho, a execução irá apresentar os eventos que ocorrem, os testes executados por cada nodo e quais as informações obtidas em cada teste, outra informação importante é o número de testes executados para diagnosticar um evento.

Algoritmo Broadcast

O TP3 foi construído sobre o Algoritmo VCube, construído no TP2. A novidade, é que a partir de um ponto determinado, a partir do arquivo de entrada com a regra B, um nodo inicia um Broadcast para a difusão de informações entre todos os nodos do sistema. A distribuição de mensagens é feita utilizando a tabela CiS, de forma que cada nodo que recebe a mensagem transmite para o primeiro nodo sem falha de cada próximo Cluster. Para implementar isso, foi criado um novo evento na SMPL, chamado BROADCAST. Este evento indica que o nodo recebeu uma mensagem e que pode precisar repeti-la para outros nodos.

Geral
